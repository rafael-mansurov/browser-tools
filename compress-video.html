Вопросы

Частые вопросы

Что значит 100% без перекодирования?

FFmpeg копирует видео- и аудиопотоки как есть и только перепаковывает файл (например, faststart для MP4 — удобнее для веба). Картинка и звук не пересчитываются, поэтому качество не меняется. Если контейнер или кодеки не позволяют просто скопировать потоки, инструмент один раз перекодирует ролик в высоком качестве (CRF 18).

Видео уходит на сервер?

Нет. Всё выполняется в твоём браузере через WebAssembly. Интернет нужен, пока страница один раз скачивает движок с CDN (при открытии страницы).

Открыл HTML двойным кликом — не работает

Страница должна открываться по адресу http://… (локальный сервер или сайт), а не как file://…. Иначе браузер режет Web Worker. Из папки проекта: python3 -m http.server 8080 → в адресной строке http://localhost:8080/compress-video.html.

Почему при открытии страницы идёт загрузка?

Сразу после открытия страницы браузер скачивает wasm-модуль FFmpeg (десятки мегабайт) — как модель на странице с водяными знаками. Дальше файл кэшируется. Длинные ролики в высоком разрешении обрабатываются дольше — это нормально для работы целиком на устройстве.
